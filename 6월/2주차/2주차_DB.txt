2026/06/08 ~ 2026/06/12 주간 배운 내용 정리 - DB

DB

- 데이터베이스 알아보기 핵심 포인트		/ 01-1(01-2는 SQL 설치 방법으로 생략)
	1. 데이터베이스는 데이터의 집합이며, DBMS는 데이터베이스를 운영/관리하는 프로그램을 말한다.

	2. 테이블은 데이터베이스의 최소 단위로, 하나 이상의 열과 행으로 구성되어 있다.

	3. SQL은 데이터베이스를 구축, 관리하고 활용하기 위해서 사용되는 언어이다.


- 데이터베이스 모델링 핵심 포인트		/ 02-1
	1. 프로젝트란 현실 세계를 컴퓨터 시스템으로 옮겨놓는 일련의 과정이다.

	2. 폭포수 모델은 소프트웨어 개발 단계 중 하나로, 이름 그대로 폭포가 떨어지듯 개발 단계가 진행된다.

	3. 데이터베이스 모델링이란 현실 세계에서 사용되는 작업이나 사물들을 DBMS의 테이블로 옮기기 위한 과정이다.


- 데이터베이스  시작부터 끝까지 핵심 포인트		/ 02-2
	1. 스키마는 MySQL 안의 데이터베이스를 말한다. 데이터베이스와 동일한 용어라고 생각하면 된다.

	2. 데이터 형식은 문자형(CHAR), 정수형(INT) 등과 같이 열에 저장될 데이터의 형식을 말한다.

	3. 예약어는 SELECT, FROM, WHERE 와 같은 기존에 약솓된 SQL 이다.

	4. 기본 키는 열에 지정하며, 각 행을 구분하는 유일한 값이다.


- 데이터베이스 개체 핵심 포인트		/ 02-3
	1. 인덱스는 책 뒤의 '찾아보기' 와 비슷한 개념이다. 데이터를 빠르게 찾을 수 있도록 도와준다.

	2. 뷰는 '바로가기 아이콘' 과 비슷한 개념이다. 한마디로 정의하면 '가상의 테이블'이라고 할 수 있다.

	3. 스토어드 프로시저는 여러 개의 SQL을 묶어주거나, 프로그래밍 기능을 제공해준다.


- SQL 기본문법 핵심 포인트		/ 03-1
	1. USE 문은 데이터베이스를 선택하는 구문으로 한 번 지정하면 계속 유지된다.

	2. SELECT ~ FROM ~ WHERE 문은 가장 기본적인 SQL 로, SELECT 다음에 열 이름, FROM 다음에 테이블 이름, WHERE 다음에 조건식이 올수 있다.
	3. 관계 연산자는 WHERE 절에서 크다/작다/같다 등을 지정하는 기호로 '<, <=, >, >=, =' 등이 있다.

	4. 논리 연산자는 관계 연산자가 2개 이상 나오면 AND, OR 등으로 참/거짓을 판별한다.

	5. LIKE는 문자열 비교 시 모두 허용할 때는 %를, 하나로 지정할 때는 _를 사용한다.
