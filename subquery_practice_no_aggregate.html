이 문서의 학습 범위

이 문서는 아직 집계 함수를 배우지 않은 학습자를 기준으로 작성했습니다.

  • 사용하지 않는 문법: COUNT(), SUM(), AVG(), MAX(), MIN(), GROUP BY

서브쿼리 핵심 개념

서브쿼리는 SQL문 안에 들어 있는 또 다른 SELECT문입니다. 보통 괄호 ( ) 안에 작성합니다.

1단계괄호 안쪽 SELECT가 먼저 실행됩니다.
2단계서브쿼리 결과가 바깥쪽 WHERE 조건에 들어갑니다.
3단계조건을 만족하는 행만 최종 조회됩니다.

=와 IN의 차이

문법사용 상황예시 의미
=서브쿼리 결과가 1개일 때트와이스의 지역 하나와 같은 지역
IN서브쿼리 결과가 여러 개일 수 있을 때디지털 상품을 구매한 여러 회원 ID 중 하나
NOT IN서브쿼리 결과 목록에 포함되지 않는 데이터를 찾을 때구매 기록이 있는 회원 ID 목록에 없는 회원
자주 나는 오류

= 뒤의 서브쿼리 결과가 2개 이상 나오면 오류가 날 수 있습니다. 결과가 여러 개 나올 가능성이 있으면 IN을 사용합니다.

1번 기본 비교 서브쿼리

트와이스와 같은 지역에 사는 그룹의 이름, 지역, 평균키를 조회하시오.

한 그룹의 지역 값을 먼저 구한 뒤, 그 지역과 같은 회원을 찾는 문제입니다.

① 먼저 생각할 순서

  1. 먼저 트와이스가 어느 지역에 사는지 찾아야 합니다.
  2. 그 결과로 나온 지역 값을 바깥쪽 SELECT문의 WHERE 조건에서 사용합니다.
  3. 지역은 한 그룹당 하나만 나오므로 = 비교 연산자를 사용할 수 있습니다.

② 서브쿼리만 먼저 실행해보기

SELECT addr
FROM member
WHERE mem_name = '트와이스';

③ 서브쿼리 결과 해석

서울

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT mem_name, addr, height
FROM member
WHERE addr = (
    SELECT addr
    FROM member
    WHERE mem_name = '트와이스'
);

⑤ 줄별 설명

SQL 부분설명
SELECT mem_name, addr, height최종 화면에 그룹 이름, 지역, 평균키를 보여줍니다.
FROM member회원 정보가 들어 있는 member 테이블에서 조회합니다.
WHERE addr = (...)지역이 서브쿼리 결과와 같은 행만 남깁니다.
SELECT addr FROM member WHERE mem_name = '트와이스'트와이스의 지역을 먼저 구합니다.

⑥ 예상 결과 예시

DB 도구에 따라 출력 순서는 다를 수 있습니다.

mem_nameaddrheight
트와이스서울167
오마이걸서울160
소녀시대서울168

강의 멘트

학생에게는 ‘트와이스의 addr 값을 먼저 구하고, 그 값이 서울이면 addr이 서울인 그룹을 다시 찾는다’고 설명하면 좋습니다.

2번 숫자 비교 서브쿼리

트와이스보다 평균키가 큰 그룹의 이름과 평균키를 조회하시오.

서브쿼리로 기준 숫자 하나를 구한 뒤, 바깥쪽 쿼리에서 `>` 연산자로 비교하는 문제입니다.

① 먼저 생각할 순서

  1. 먼저 트와이스의 평균키를 조회합니다.
  2. 그 키보다 큰 그룹만 바깥쪽 쿼리에서 조회합니다.
  3. 트와이스의 키는 하나만 나오기 때문에 > 비교가 가능합니다.

② 서브쿼리만 먼저 실행해보기

SELECT height
FROM member
WHERE mem_name = '트와이스';

③ 서브쿼리 결과 해석

167

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT mem_name, height
FROM member
WHERE height > (
    SELECT height
    FROM member
    WHERE mem_name = '트와이스'
);

⑤ 줄별 설명

SQL 부분설명
SELECT mem_name, height그룹 이름과 평균키만 출력합니다.
WHERE height > (...)평균키가 서브쿼리 결과보다 큰 행만 조회합니다.
SELECT height FROM member WHERE mem_name = '트와이스'트와이스의 평균키 167을 먼저 구합니다.

⑥ 예상 결과 예시

DB 도구에 따라 출력 순서는 다를 수 있습니다.

mem_nameheight
소녀시대168
강의 멘트

여기서 `이상`과 `초과`를 구분해 주세요. 문제는 ‘큰’이므로 `>`이고, ‘크거나 같은’이면 `>=`입니다.

3번 날짜 비교 서브쿼리

소녀시대보다 늦게 데뷔한 그룹의 이름과 데뷔일을 조회하시오.

날짜도 숫자처럼 비교할 수 있습니다. 더 뒤의 날짜는 더 늦은 날짜로 판단합니다.

① 먼저 생각할 순서

  1. 먼저 소녀시대의 데뷔일을 조회합니다.
  2. 그 데뷔일보다 큰 날짜를 가진 그룹을 찾습니다.
  3. 날짜 비교에서는 >가 ‘더 늦은 날짜’를 의미합니다.

② 서브쿼리만 먼저 실행해보기

FROM member
WHERE mem_name = '소녀시대';

4번 문자 비교 서브쿼리

에이핑크와 같은 전화 국번을 가진 그룹의 이름, 전화 국번, 전화번호 뒷자리를 조회하시오.

문자 데이터도 `=`로 비교할 수 있습니다. 전화 국번은 `phone1` 컬럼에 저장되어 있습니다.

① 먼저 생각할 순서

  1. 먼저 에이핑크의 전화 국번 phone1 값을 찾습니다.
  2. 그 값과 같은 phone1을 가진 그룹을 조회합니다.
  3. 전화 국번은 문자처럼 저장되어 있으므로 따옴표가 붙은 값이라고 생각하면 됩니다.

② 서브쿼리만 먼저 실행해보기

SELECT phone1
FROM member
WHERE mem_name = '에이핑크';

③ 서브쿼리 결과 해석

④ 최종 풀이 SQL

SELECT mem_name, phone1, phone2
FROM member

SQL 부분설명

5번 IN 서브쿼리

디지털 상품을 구매한 적이 있는 그룹의 모든 회원 정보를 조회하시오.

서브쿼리 결과가 여러 개 나올 수 있으므로 `=`가 아니라 `IN`을 사용해야 하는 문제입니다.

① 먼저 생각할 순서

  1. 먼저 buy 테이블에서 디지털 상품을 구매한 회원 ID들을 찾습니다.
  2. 디지털 상품을 구매한 회원은 여러 명일 수 있습니다.
  3. 여러 개의 ID 목록 중 하나에 해당하는 회원을 찾아야 하므로 IN을 사용합니다.

② 서브쿼리만 먼저 실행해보기

SELECT mem_id
FROM buy
WHERE group_name = '디지털';

③ 서브쿼리 결과 해석

BLK, APN, MMU 등이 나올 수 있음

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT *
FROM member
WHERE mem_id IN (
    SELECT mem_id
    FROM buy
    WHERE group_name = '디지털'
);

⑤ 줄별 설명

SQL 부분설명
SELECT * FROM membermember 테이블의 모든 열 정보를 출력합니다.
WHERE mem_id IN (...)회원 ID가 서브쿼리 결과 목록 안에 있는지 확인합니다.
SELECT mem_id FROM buy WHERE group_name = '디지털'디지털 상품을 구매한 회원 ID 목록을 먼저 만듭니다.

⑥ 예상 결과 예시

DB 도구에 따라 출력 순서는 다를 수 있습니다.

여기서 핵심은 `=`와 `IN`의 차이입니다. 서브쿼리 결과가 여러 개일 수 있으면 `IN`을 쓰는 습관을 들이게 해주세요.

6번 NOT IN 서브쿼리

디지털 상품을 구매한 적이 없는 그룹의 아이디와 그룹 이름을 조회하시오.

5번의 반대 문제입니다. 목록 안에 포함되지 않은 데이터를 찾을 때 `NOT IN`을 사용합니다.

① 먼저 생각할 순서

  1. 먼저 디지털 상품을 구매한 회원 ID 목록을 만듭니다.
  2. 그 목록에 들어 있지 않은 회원을 member 테이블에서 찾습니다.
  3. ‘없는’, ‘하지 않은’, ‘제외한’이라는 말이 나오면 NOT IN을 생각할 수 있습니다.

② 서브쿼리만 먼저 실행해보기

FROM buy
WHERE group_name = '디지털';

③ 서브쿼리 결과 해석

BLK, APN, MMU 등이 나올 수 있음

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT mem_id, mem_name
FROM member
WHERE mem_id NOT IN (

    FROM buy
    WHERE group_name = '디지털'
);

SQL 부분설명

7번 중첩 서브쿼리

에이핑크가 구매한 상품과 같은 상품을 구매한 모든 구매 내역을 조회하시오.

회원 이름에서 회원 ID를 찾고, 그 ID로 구매 상품명을 찾고, 같은 상품명을 가진 구매 내역을 다시 찾는 문제입니다.

① 먼저 생각할 순서

  1. buy 테이블에는 그룹 이름이 없고 mem_id만 있습니다.
  2. 따라서 먼저 member 테이블에서 에이핑크의 mem_id를 찾아야 합니다.
  3. 그 다음 buy 테이블에서 에이핑크가 구매한 상품명을 찾습니다.
  4. 마지막으로 같은 상품명을 가진 모든 구매 내역을 조회합니다.

② 서브쿼리만 먼저 실행해보기

SELECT mem_id
FROM member
WHERE mem_name = '에이핑크';

SELECT prod_name
FROM buy
WHERE mem_id = 'APN';

③ 서브쿼리 결과 해석

APN → 아이폰, 혼공SQL, 청바지

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT *
FROM buy
WHERE prod_name IN (
    SELECT prod_name
    FROM buy
    WHERE mem_id = (
        SELECT mem_id
        FROM member
        WHERE mem_name = '에이핑크'
    )
);

⑤ 줄별 설명

SQL 부분설명

8번 분류 기준 IN 서브쿼리

상품명 자체가 아니라 상품 분류를 기준으로 같은 종류의 구매 내역을 찾는 문제입니다.

① 먼저 생각할 순서

  1. 먼저 에이핑크의 mem_id를 찾습니다.
  2. 그 ID로 에이핑크가 구매한 상품들의 group_name을 찾습니다.

SELECT group_name

9번 3단계 중첩 서브쿼리

상품명 → 구매한 회원 ID → 그 회원의 지역 → 같은 지역 회원 조회 순서로 진행되는 문제입니다.

① 먼저 생각할 순서

  1. 먼저 buy 테이블에서 청바지를 구매한 회원 ID를 찾습니다.
  2. 그 회원 ID들을 member 테이블에서 찾아 지역을 구합니다.
  3. 마지막으로 그 지역에 사는 모든 그룹을 조회합니다.
  4. 청바지를 구매한 그룹이 여러 명일 수 있으므로 IN이 필요합니다.

② 서브쿼리만 먼저 실행해보기

SELECT mem_id
FROM buy
WHERE prod_name = '청바지';

SELECT addr
FROM member
WHERE mem_id IN ('BLK', 'APN');

BLK, APN → 경남, 경기

10번 조건 2개 + 서브쿼리 2개

블랙핑크보다 인원수가 많고, 구매 기록이 없는 그룹의 이름과 인원수를 조회하시오.

하나의 SQL에서 두 가지 조건을 동시에 검사하는 문제입니다. 숫자 비교 서브쿼리와 `NOT IN` 서브쿼리를 함께 사용합니다.

① 먼저 생각할 순서

  1. 첫 번째 조건은 블랙핑크보다 인원수가 많아야 한다는 것입니다.
  2. 두 번째 조건은 buy 테이블에 구매 기록이 없어야 한다는 것입니다.
  3. 두 조건을 모두 만족해야 하므로 AND로 연결합니다.

② 서브쿼리만 먼저 실행해보기

SELECT mem_number
FROM member
WHERE mem_name = '블랙핑크';

SELECT mem_id
FROM buy;

③ 서브쿼리 결과 해석

블랙핑크 인원수 4 / 구매 기록이 있는 mem_id 목록

이 결과가 바깥쪽 SQL문의 조건값으로 들어갑니다.

④ 최종 풀이 SQL

SELECT mem_name, mem_number
FROM member
WHERE mem_number > (
    SELECT mem_number
    FROM member
    WHERE mem_name = '블랙핑크'
)
AND mem_id NOT IN (
    SELECT mem_id
    FROM buy
);

⑤ 줄별 설명

SQL 부분설명
WHERE mem_number > (...)블랙핑크 인원수보다 큰 그룹만 남깁니다.
SELECT mem_number FROM member WHERE mem_name = '블랙핑크'블랙핑크의 인원수 4를 구합니다.
AND mem_id NOT IN (...)구매 기록이 있는 회원 ID 목록에 포함되지 않은 그룹만 남깁니다.
SELECT mem_id FROM buy구매 기록이 있는 회원 ID 목록을 만듭니다.

마지막 문제는 `조건1 AND 조건2` 구조입니다. 학생에게 ‘두 개의 필터를 차례대로 거른다’고 설명하면 좋습니다.

마무리 정리

  • 서브쿼리는 괄호 안쪽 SELECT부터 먼저 실행된다고 생각하면 됩니다.
  • 서브쿼리 결과가 하나면 =, 여러 개일 수 있으면 IN을 사용합니다.
  • 목록에 없는 데이터를 찾을 때는 NOT IN을 사용합니다.
  • 문제가 어려워 보이면 한 번에 풀려고 하지 말고, 서브쿼리만 먼저 실행해서 결과를 확인하면 됩니다.
